Customer Shopping Behaviour Analysis

End-to-end case study using Python, MySQL and Power BI — 3,900 transactions analysed.

Download Resume

Project Summary

This analysis examines customer shopping behaviour across product categories to identify spending patterns, subscription adoption, and opportunities for retention and revenue growth. The dataset contains 3,900 purchase records and 18 columns covering demographics, transaction details and behaviour signals.

Dataset & Key Features

  • Rows: 3,900
  • Columns: 18
  • Key fields: Age, Gender, Location, Subscription Status, Item, Category, Purchase Amount, Season, Size, Discount, Previous Purchases, Rating, Shipping Type.
  • Missing values: 37 ratings (imputed by category median during cleaning).

Work Completed

  1. Python & EDA: Data loading with pandas, df.info(), df.describe(), distribution checks and visual exploration.
  2. Cleaning: Imputed missing review ratings, standardized column names to snake_case, removed redundant promo_code column, handled outliers and duplicates.
  3. Feature engineering: Created age_group and purchase_frequency_days to support segmentation and churn analysis.
  4. MySQL: Loaded cleaned dataframe into MySQL, created schema and indexes, ran analytical queries to compute KPIs.
  5. Power BI: Connected to MySQL and built an interactive dashboard with filters, trend charts and customer segment pages.
  6. Reporting: Compiled findings into a stakeholder-ready report and PPT summarizing insights and recommendations.

Sample SQL Questions Answered

  • Revenue by gender and by age_group.
  • Subscription adoption impact: average spend of subscribers vs non-subscribers.
  • Top products per category and discount-dependent products.
  • Customer segmentation into New, Returning, Loyal using previous purchase counts.
  • Repeat buyers & likelihood to subscribe (customers with >5 purchases).

Example SQL (simplified):

SELECT age_group, SUM(purchase_amount) as revenue FROM purchases GROUP BY age_group ORDER BY revenue DESC;

Key Findings

  • Only 27% of customers are subscribers — major conversion opportunity.
  • Female customers spend slightly higher than overall average ($60.25 vs $59.76).
  • Clothing leads revenue and transaction count; Accessories are second. Footwear and Outerwear underperform.
  • Medium (45%) and Large (27%) sizes dominate demand — inventory planning should prioritize these.
  • Spring has the highest sales; Summer is the lowest.
  • Young Adults are the most profitable segment across purchase frequency and payment methods.

Recommendations

  • Implement targeted subscription campaigns for Young Adults and Middle-aged female customers with clear benefits (discounts, early access).
  • Run seasonal promotions and bundles for Footwear and Outerwear to lift demand.
  • Improve customer experience to increase review rating from 3.7 to 4.2+ through quality and returns policy improvements.
  • Optimize stock for M and L sizes and consider pre-orders for S and XL.
  • Use free-shipping thresholds (e.g. free shipping above $50) to grow average basket size.

How to reproduce

  1. Clone repository and ensure Python 3.9+, MySQL and Power BI Desktop are installed.
  2. Install packages: pip install -r requirements.txt.
  3. Run the notebook to clean data and export cleaned CSV.
  4. Import SQL script into MySQL and load cleaned data.
  5. Open Power BI, connect to MySQL, and load the provided .pbix file for dashboard visuals.

Car Sales Forecasting

Model-wise forecasting for wholesale and retail, yearly and quarterly projections using regression methods.

Download Resume

Project Summary

Forecasts were produced for each car model to estimate wholesale and retail volumes for 2026–2028. The work included data cleaning, time-series feature preparation, and linear regression grouped by model names to produce annual and quarterly forecasts.

Dataset & Challenges

  • Dataset contained model-level sales, distribution channel (wholesale/retail), quarterly splits, and financial year labels.
  • Partial nulls in quarterly wholesale/retail required careful imputation and smoothing.
  • Multiple car model naming variations were standardized during cleaning.

Methodology

  1. Aggregate historical sales to model-level yearly and quarterly time-series.
  2. Feature engineering: lag features (1–4 quarters), rolling mean, month/quarter dummies.
  3. Modeling: linear regression per model (simple, explainable) and grouped predictions for wholesale and retail separately.
  4. Validation: backtest on historical years and measure MAE / RMSE per model.
  5. Output: forecast CSVs and visualizations for Power BI integration.

Key Outputs

  • Model-wise annual forecasts for 2026–2028 for wholesale and retail.
  • Quarterly breakdowns for each model to support production planning.
  • CSV exports and Power BI visuals for dashboards and stakeholder reporting.

Business Impact

These forecasts help procurement and sales teams align production and inventory with expected demand per model and channel, reducing stock-outs and overproduction risk.

How to reproduce

HR Analytics

Attrition analysis, employee segmentation and retention strategies using combined HR datasets.

Download Resume

Project Summary

This project explores employee attrition drivers, segments employees by tenure and performance, and recommends interventions to reduce turnover. Techniques include exploratory analysis, correlation checks, and basic predictive modeling for attrition risk.

Dataset & Key Features

  • Typical fields: Employee ID, Age, Department, Tenure, Salary Band, Performance Score, Last Promotion, Attrition Flag.
  • Data preparation: categorical encoding, handling missing promotion dates, tenure calculation.

Approach

  1. EDA to identify high-attrition departments and tenure buckets.
  2. Feature engineering: tenure groups, promotion lag, performance trends.
  3. Modeling: logistic regression to flag high-risk employees and validate with ROC-AUC.
  4. Recommendations: targeted retention programs, upskilling and promotion cadence improvements.

Key Findings

  • High attrition observed in early-tenure ( <12 months) and specific departments.
  • Low promotion rates and stagnant performance trends correlate with attrition.
  • Employees with clear career progression and mentoring are less likely to leave.

Recommendations

  • Implement onboarding mentorship and early-career engagement for hires in first 12 months.
  • Track promotion lag and create fast-track programs for high performers.
  • Use attrition risk scoring to proactively retain high-value employees.

How to reproduce

  1. Run the HR notebooks to reproduce EDA and model artifacts.
  2. Connect outputs to Power BI for dashboards showing attrition by department and tenure.
